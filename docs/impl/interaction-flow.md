# 交互动线专项：选图 / 启用 / 反馈规范

> 专项目标：把「选图」与「启用壁纸」拆成两个独立配置；任何操作必须有就近、即时反馈；分屏选图必须有预览，且不必滚到页面底部才能生效。  
> 本文是 **UI/交互统一规范**，实现以本文为准；与旧「点底部应用才生效」行为冲突时，以本文为准。

---

## 1. 问题回顾（改造前）

| 现象 | 根因 |
|------|------|
| 选完图没反应 | 选图只写路径，无预览刷新文案/无就近操作 |
| 只能点底部「应用壁纸」 | 选图与启用耦合在一个远端按钮 |
| 点应用仍像没反馈 | busy/结果提示不够醒目，或失败被吞 |
| 想停止只能删图 | 没有独立的「停用」；删图成了唯一退出手段 |
| 分屏无预览、链路远 | 分屏行只有路径文字；操作入口在页底 |

---

## 2. 核心模型：两套配置

| 配置 | Key | 含义 | 改它会不会立刻铺桌面 |
|------|-----|------|----------------------|
| **选图** | `imagePath` / `perDisplayMap` | 用户准备用哪张图 | **未启用：否**；**已启用：是**（立即重铺） |
| **启用** | `wallpaperEnabled` | 是否允许 Ink Paper 把壁纸铺到桌面 | **是**（开→按偏好应用；关→停止 overlay） |

补充：

- `preferredMode`：自动 / 强制系统 / 强制底层 —— 只在 **已启用** 时参与决策。
- `activeMode`：运行时实际模式（system / overlay / nil），由 ModeEngine 维护，不是用户直接点的开关。
- `overlayEnabled`：运行态镜像（底层窗是否在），由引擎写入；UI 不直接当主开关。

### 状态机（简化）

```
未选图 + 未启用  →  选图  →  已选图 + 未启用（仅预览）
                         ↓ 打开「启用壁纸」
                    已选图 + 已启用（桌面生效）
                         ↓ 关闭「启用壁纸」
                    已选图 + 未启用（图保留，桌面停用）
                         ↓ 移除图片
                    未选图 + 未启用
```

规则：

1. **未选有效图时，不允许打开启用**（开关 disabled + 说明原因）。
2. **停用不得要求用户删图**；删图是资源管理，停用是运行控制。
3. **已启用时改图 / 改缩放 / 改分屏**：直接重铺桌面，**不需要**再点「更新到桌面」。
4. **启动时**：仅当 `wallpaperEnabled == true` 且有有效图，才自动 `applyPreferredMode`。

## 3. 统一反馈规范

### 3.1 三级反馈（必须同时满足可感知）

| 级别 | 形式 | 何时 |
|------|------|------|
| L1 就近 | 按钮旁/行内文案、缩略图变化、开关状态 | 每次点击后 100ms 内可见 |
| L2 顶栏 | 全局 statusBar（busy / 成功 / 失败） | 所有异步桌面操作 |
| L3 忙态 | 中央 Progress + 相关控件 disabled | `isBusy == true` |

禁止：静默成功、只改内存无任何 UI 变化。

### 3.2 文案模板

| 动作 | 成功文案示例 | 失败 |
|------|--------------|------|
| 选全局图 | 未启用：「已选择…（尚未启用）」；已启用：立即应用到桌面 | 弹 alert |
| 选分屏图 | 同上 | 弹 alert |
| 改用兜底 | 已改为使用兜底图…（已启用则立即重铺） | — |
| 仅原生壁纸 | 已设为仅原生壁纸…（已启用则立即重铺） | — |
| 移除全局图 | 已移除…（仍有可铺图且已启用则重铺，否则停用） | — |
| 打开启用 | 正在启用壁纸… → 已启用：底层窗口/系统壁纸 | lastError + 顶栏红字 |
| 关闭启用 | 已停用壁纸（图片仍保留） | — |

### 3.3 操作就近原则

- 选图控件旁必须有：**预览** + **移除/改用兜底/仅原生**（已启用时改完即生效，无单独「更新」按钮）。
- 启用开关放在壁纸页 **最顶部固定区**，不埋在 Form 底部。
- 分屏每一行自包含：缩略图、选图、改用兜底、仅原生壁纸。

## 4. 壁纸页信息架构（规范布局）

自上而下：

1. **启用区（固定）**  
   - Toggle「启用壁纸」  
   - 状态徽章：未启用 / 启用中 / 运行中（系统|底层） / 失败  
   - 辅助说明一行
2. **分屏开关**
3. **分屏列表**（开启时）：每行预览 + 选图 / 改用兜底 / 仅原生壁纸；可刷新显示器列表  
4. **全局图 / 兜底图**：大预览 + 选图/移除  
5. **缩放策略**（已启用时改完即重铺）  
6. **危险操作**：清除全部已选图（不影响「启用」开关本身，但无图时应自动停用）

分屏管理全部在壁纸页完成，无独立「显示器」Tab。

### 分屏三态

| 操作 | 配置 | 含义 |
|------|------|------|
| 选择图片 | `perDisplayMap[id] = path`，移出 native 集合 | 该屏独立图 |
| 改用兜底图 | 删除 map 键，移出 native 集合 | 回退全局 `imagePath` |
| 仅原生壁纸 | 加入 `perDisplayNativeIDs`（保留 map 路径） | 不铺覆盖层、不写系统壁纸 |
| 恢复覆盖 | 移出 native 集合 | 恢复分屏图或兜底图覆盖 |

## 5. ModeEngine 语义（与 UI 对齐）

| API | 行为 |
|-----|------|
| `enableWallpaperAsync()` | `wallpaperEnabled=true` → `applyPreferredMode` |
| `disableWallpaperAsync()` | 停 overlay；`wallpaperEnabled=false`；`activeMode` 对 overlay 清 nil；**不删图** |
| `updateDesktopAsync()` | 要求已启用；等价于按当前偏好/当前模式重铺 |
| `bootstrap()` | 仅 `wallpaperEnabled && 有图` 时自动应用；否则只做健康检查 |

强制「切换到系统/底层」按钮：若未启用，先视为启用意图（打开 `wallpaperEnabled`）再切换；或提示先启用——**规范采用：模式页操作隐含启用**。

## 6. 验收清单

- [ ] 未启用时选图：桌面不变，文案「尚未启用」
- [ ] 启用后桌面变化且有 busy→成功反馈
- [ ] 已启用时选图/换图/改缩放：**立即**重铺，无「更新到桌面」按钮
- [ ] 停用后底层窗消失，图片配置仍在，可再次启用
- [ ] 分屏每行有缩略图；选图不必滚到底部
- [ ] 「仅原生壁纸」不回退兜底图；该屏无 overlay 窗
- [ ] 「改用兜底图」与「仅原生」语义不同
- [ ] 清除全部图时自动停用
- [ ] 诊断/模式页文案与上述状态一致
- [ ] 无独立「显示器」Tab

| 规范项 | 实现 |

| 启动不强制铺桌面 | `bootstrap` 仅在已启用时应用 |
